Use este pacote como projeto final de um site estático para GitHub Pages.

Tarefas:
1. Criar um repositório no GitHub.
2. Subir todos os arquivos exatamente na raiz do repositório.
3. Manter o site como HTML/CSS/JS estático, sem build obrigatório.
4. Conferir se `data/PP23.dxf`, `assets/openblast-logo.png` e `assets/default-report-logo.png` foram enviados.
5. Ativar GitHub Pages em `Settings > Pages > Deploy from a branch > main > /root`.
6. Não adicionar citações textuais à marca do logo padrão na interface. Manter o visual técnico, limpo e corporativo.
7. Confirmar se o botão de exportar PDF funciona no navegador após a publicação.

Critérios de aceite:
- A página abre pelo GitHub Pages.
- O DXF padrão é carregado automaticamente.
- O usuário consegue editar o nome do fogo analisado.
- O usuário consegue inserir outro logo no relatório.
- Os gráficos de ângulo e direção aparecem.
- O relatório exporta em PDF.
